# 飞书登录流程

Apex 支持两种登录场景，共用同一套后端鉴权体系（JWT）。

---

## 场景一：飞书客户端内（H5 静默登录）

用户从飞书工作台或消息卡片点击进入，飞书 JSSDK 静默获取临时 code，POST 给后端换 JWT，全程用户无感知。

### 前置配置（飞书开放平台）

| 配置项 | 值 |
|--------|-----|
| 应用类型 | 企业自建应用 |
| 功能 | 网页应用（H5） |
| 网页 URL（入口） | `https://your-domain.com/api/auth/feishu/launch` |
| 可信域名 | `your-domain.com` |
| 所需权限 | `contact:user.base:readonly` |

### 流程图

```
用户点击飞书工作台应用
        │
        ▼
飞书 WebView 加载 H5 入口
https://your-domain.com/api/auth/feishu/launch
        │ 302
        ▼
前端 /matrix 页面
        │
        ▼
FeishuGuard 检测到飞书环境 → 透传
        │
        ▼
RequireAuth 检查 localStorage token
  ├─ 有效 token → GET /api/auth/me → 渲染应用
  └─ 无 token → 调用飞书 JSSDK
                 window.tt.requestAccess({ appID })
                 → 静默获得临时 code（5 分钟有效）
                        │
                        ▼
               POST /api/auth/feishu/code2token { code }
                        │
                        ▼
               后端换取用户信息 → 签发 JWT
                        │
                        ▼
               前端存 token → 渲染应用
```

## 场景二：外部浏览器（网页 OAuth 授权）

用户在普通浏览器中打开应用链接，`FeishuGuard` 检测到非飞书环境，自动跳转飞书网页扫码授权页，授权完成后回调带 JWT 落地前端。

### 前置配置（飞书开放平台 → 安全设置 → 重定向 URL）

需添加白名单：

```
https://your-domain.com/api/auth/feishu/callback
```

### 流程图

```
外部浏览器访问 https://your-domain.com
        │
        ▼
FeishuGuard 检测到非飞书环境
        │ 自动跳转（window.location）
        ▼
GET /api/auth/feishu/web-login
        │ 302
        ▼
飞书网页授权页
https://open.feishu.cn/open-apis/authen/v1/authorize?...
        │
        │ 用户扫码 / 账密登录
        ▼
GET /api/auth/feishu/callback?code=xxx&state=web_login
        │
        ▼
后端换取用户信息 → 签发 JWT
        │ 302，token 放在 URL fragment
        ▼
前端 /auth-callback#token=<JWT>
        │
        ▼
AuthCallbackPage 从 fragment 读取 token
→ 存入 localStorage → 跳转 /matrix
```

> **为什么用 URL fragment 传 token？**
> fragment（`#` 后的部分）不会发送到服务器，不会出现在 Nginx/Caddy 日志中，比 query string 更安全。

## 本地开发 Mock 流程

`import.meta.env.DEV === true` 时，`FeishuGuard` 直接透传，`RequireAuth` 走 mock 登录：

```
浏览器 dev 模式访问
        │
        ▼
FeishuGuard DEV 豁免 → 透传
        │
        ▼
RequireAuth 检测到非飞书环境
        │
        ▼
POST /api/auth/mock-login
        │
        ▼
后端签发包含 mock 用户信息的真实 JWT
        │
        ▼
前端存 token → GET /api/auth/me → 渲染应用
（与生产静默流程完全一致，仅跳过飞书 API 调用）
```

---

## Token 失效处理

```
前端请求收到 401
        │
        ▼
清除 localStorage access_token
        │
        ├─ 飞书客户端内 → 重走 JSSDK 静默登录（用户无感知）
        └─ 外部浏览器   → 重走网页 OAuth 跳转授权
```

## 后端接口一览

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/auth/feishu/code2token` | 飞书客户端 JSSDK code 换 JWT | 无 |
| GET  | `/api/auth/feishu/launch` | 飞书工作台入口跳转，302 到前端 | 无 |
| GET  | `/api/auth/feishu/web-login` | 外部浏览器发起网页 OAuth，302 到飞书授权页 | 无 |
| GET  | `/api/auth/feishu/callback` | 网页 OAuth 回调，换 JWT 后 302 到前端 `/auth-callback` | 无 |
| GET  | `/api/auth/me` | 获取当前登录用户信息 | JWT |
| POST | `/api/auth/mock-login` | Mock 登录（仅本地开发） | 无 |

## 环境变量

```bash
# 后端 .env.production
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
FEISHU_REDIRECT_URI=https://your-domain.com/api/auth/feishu/callback
FRONTEND_URL=https://your-domain.com
JWT_SECRET=your-random-secret-string
JWT_EXPIRE_MINUTES=1440   # 24 小时

# 前端 .env / .env.production
VITE_FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
VITE_API_BASE_URL=https://your-domain.com
```

## 注意事项

- JSSDK code **一次性使用**，5 分钟内有效，用完即失效
- 网页 OAuth code 同样一次性，回调后立即消费
- JWT 有效期 24 小时，过期后自动重走对应场景的授权流程
- 生产环境必须 HTTPS，可信域名和重定向 URL 需与实际部署域名完全一致
- `mock-login` 接口无鉴权，**仅限本地开发使用**，生产环境不要暴露或调用
