# 飞书登录流程（H5 网页应用）

## 概述

用户从飞书工作台或消息卡片点击进入 Apex H5 应用，飞书负责鉴权，系统无独立登录页。前端通过飞书 JSSDK 静默获取临时授权码（code），POST 给后端换取 JWT，完成免密登录，全程用户无感知。

---

## 前置条件（飞书开放平台配置）

| 配置项 | 值 |
|--------|-----|
| 应用类型 | 企业自建应用 |
| 功能 | 网页应用（H5） |
| 网页 URL（入口） | `https://your-domain.com` |
| 可信域名 | `https://your-domain.com` |
| 所需权限 | `contact:user.base:readonly` |

> 开发阶段可用 cloudflared 临时域名替代，生产环境替换为正式域名后重新配置即可。

## 完整登录流程

```
┌──────────────────────────────────────────────────────────────────┐
│                      用户侧（飞书客户端）                           │
└──────────────────────────────────────────────────────────────────┘
          │
          │  1. 点击工作台应用图标 或 消息卡片按钮
          ▼
┌──────────────────────────────────────────────────────────────────┐
│                      飞书平台                                      │
│  在 WebView 中打开 H5 入口 URL                                     │
│  https://your-domain.com/                                        │
└──────────────────────────────────────────────────────────────────┘
          │
          │  2. WebView 加载 H5 页面
          ▼
┌──────────────────────────────────────────────────────────────────┐
│                      前端 H5（React）                              │
│                                                                  │
│  RequireAuth 组件挂载，检查 localStorage：                          │
│                                                                  │
│  ┌──────────────────┐      ┌─────────────────────────────────┐  │
│  │ 有有效 JWT token  │      │ 无 token 或 token 已过期          │  │
│  └────────┬─────────┘      └────────────┬────────────────────┘  │
│           │                             │                        │
│           ▼                             ▼                        │
│    GET /api/auth/me             调用飞书 JSSDK                    │
│    补全用户信息                   window.h5sdk.ready(() => {      │
│    直接渲染应用                     h5sdk.getAuthCode({           │
│                                     appId: 'cli_xxx'            │
│                                   })                            │
│                                 })                              │
│                                 → 静默获得临时 code（5 分钟有效）  │
└──────────────────────────────────────────────────────────────────┘
          │
          │  3. 前端携带 code 请求后端
          │  POST /api/auth/feishu/code2token   { "code": "xxx" }
          ▼
┌──────────────────────────────────────────────────────────────────┐
│                      后端（FastAPI）                               │
│                                                                  │
│  Step 1  获取应用级 tenant_access_token                            │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ POST https://open.feishu.cn/open-apis/auth/v3/           │    │
│  │      tenant_access_token/internal                        │    │
│  │ Body: { app_id, app_secret }                            │    │
│  │ → 返回 tenant_access_token                               │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  Step 2  用 code 换取用户 access_token 及用户信息                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ POST https://open.feishu.cn/open-apis/authen/v1/         │    │
│  │      oidc/access_token                                   │    │
│  │ Header: Authorization: Bearer <tenant_access_token>     │    │
│  │ Body: { grant_type: "authorization_code", code }        │    │
│  │ → 返回 open_id / name / avatar_url / email              │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  Step 3  签发 JWT                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │ payload: { open_id, name, avatar_url, email, exp }      │    │
│  │ 返回 { access_token, token_type: "bearer" }             │    │
│  └─────────────────────────────────────────────────────────┘    │
└──────────────────────────────────────────────────────────────────┘
          │
          │  4. 返回 JWT
          ▼
┌──────────────────────────────────────────────────────────────────┐
│                      前端 H5（React）                              │
│                                                                  │
│  localStorage.setItem('access_token', jwt)                       │
│  GET /api/auth/me → 填充用户信息到 Zustand store                   │
│  渲染应用（Dashboard）                                             │
└──────────────────────────────────────────────────────────────────┘
          │
          │  5. 后续所有 API 请求
          ▼
┌──────────────────────────────────────────────────────────────────┐
│  请求头统一携带 Authorization: Bearer <JWT>                        │
│  后端通过 get_current_user 依赖验证 JWT → 注入用户信息 → 返回数据    │
└──────────────────────────────────────────────────────────────────┘
```

## Token 失效处理

```
前端请求收到 401 响应
        │
        ▼
清除 localStorage access_token
        │
        ▼
重新调用 window.h5sdk.getAuthCode() 静默获取新 code
→ POST /api/auth/feishu/code2token 换取新 JWT
（用户无感知，自动静默刷新）
        │
        ▼
若 getAuthCode 也失败（用户离开飞书环境）
        │
        ▼
页面显示"登录失败，请重新打开应用"
```

## 本地开发 Mock 流程

本地浏览器中不存在飞书客户端环境（`window.h5sdk` 为 `undefined`），通过调用后端 mock 接口模拟完整登录流程：

```
前端检测到 import.meta.env.DEV === true
        │
        ▼
POST /api/auth/mock-login（后端 DEBUG=true 时可用）
        │
        ▼
后端签发包含 mock 用户信息的真实 JWT
        │
        ▼
前端存 token → GET /api/auth/me → 渲染应用
（与生产静默流程完全一致，仅跳过飞书 API 调用）
```

### 前端 — 静默登录方法

```typescript
// src/services/auth.ts

// 生产环境：飞书 JSSDK 静默获取 code，换取 JWT
export function silentLogin(): Promise<string> {
  return new Promise((resolve, reject) => {
    window.h5sdk.ready(() => {
      window.h5sdk.getAuthCode({
        appId: import.meta.env.VITE_FEISHU_APP_ID,
        success: async ({ code }) => {
          const res = await request.post('/auth/feishu/code2token', { code })
          resolve(res.data.data.access_token)
        },
        fail: reject,
      })
    })
  })
}

// 开发环境：调后端 mock-login，返回真实 JWT
export async function mockSilentLogin(): Promise<string> {
  const res = await request.post('/auth/mock-login')
  return res.data.data.access_token
}
```

## 注意事项

- `getAuthCode` 获取的 code **一次性使用**，5 分钟内有效，用完即失效
- JWT 有效期建议 24 小时，过期后由 RequireAuth 自动静默刷新
- 飞书 JSSDK（`window.h5sdk`）只在飞书 WebView 环境下有效，本地浏览器调试走 mock 流程
- 生产环境必须 HTTPS，可信域名需与实际部署域名完全一致
- 生产环境 `DEBUG` 必须设为 `false`，否则任何人都可调用 `mock-login` 绕过鉴权
