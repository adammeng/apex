# Apex 前端测试方案

## 1. 文档目标

本方案用于规范 Apex 前端在 V1 阶段的测试范围、测试方法、执行步骤与通过标准，覆盖 PRD 中本期纳入范围的前端能力：

- 飞书访问入口与客户端引导
- 飞书静默登录与登录态恢复
- 树状筛选与筛选联动
- 靶点组合竞争矩阵视图
- 靶点研发进展泳道图视图
- Excel 导出触发能力

## 3. 测试目标

- 验证前端路由、页面和守卫逻辑与 PRD 的 V1 范围一致
- 验证飞书环境与非飞书环境的访问分流符合设计
- 验证矩阵页、泳道页的主要筛选、展示、导出交互可用
- 验证生产构建可成功产出静态资源，支持 Docker / Nginx 部署

## 4. 测试范围

### 4.1 In Scope

- 路由：`/`、`/matrix`、`/pipeline`
- 守卫：`FeishuGuard`、`RequireAuth`
- 认证交互：飞书静默登录、开发环境 mock 登录、登录失败信息展示
- 矩阵页：疾病树、阶段筛选、隐藏无组合开关、tooltip 详情、导出
- 泳道页：疾病单选、靶点多选、泳道卡片渲染、导出
- 打包构建：`npm run build`

### 4.2 Out of Scope

- 数据总览页
- 竞争格局页
- 前端视觉回归自动化
- 浏览器兼容性矩阵的全量回归
- 性能压测和弱网专项压测

## 9. 风险与补测建议

- 当前前端缺少单元测试与 E2E 自动化，主要依赖构建检查与手工联调
- 飞书 H5 SDK 行为受真实飞书应用配置影响，应在测试环境和生产环境各执行一次完整回归
- 生产构建产物较大，建议后续对矩阵页/泳道页做按路由拆包与 `manualChunks` 优化
