### 2.4 本地性能压测结果

PRD 7.1 提出的关键后端相关指标：

- 数据查询：复杂多条件筛选 `< 5s`
- 并发能力：`>= 100` 并发用户

本轮压测环境：

- 本地服务地址：`http://127.0.0.1:8000`
- 数据源：`backend/parquet/` 本地样本
- Redis：本地 Homebrew Redis，压测前清空 `apex:*` 缓存键
- 工具：`curl`、`ab`
- 样本：
  - 矩阵查询：`non-small cell lung cancer (NSCLC)`
  - 泳道查询：`Asthma`

本轮执行结果：

| 场景 | 结果 | 结论 |
| --- | --- | --- |
| 矩阵查询冷启动 | `0.025s` | 达到 `< 5s` |
| 泳道查询冷启动 | `0.014s` | 达到 `< 5s` |
| 矩阵查询缓存命中中位数 | `0.004s` | 缓存收益明显 |
| 泳道查询缓存命中中位数 | `0.007s` | 缓存收益明显 |
| 矩阵查询 `n=100,c=50` | `0` 失败，P95 `891ms` | 达到 `< 5s` |
| 矩阵查询 `n=200,c=100` | `0` 失败，P95 `360ms` | 达到 `>=100` 并发目标 |
| 泳道查询 `n=100,c=50` | `0` 失败，P95 `355ms` | 达到 `< 5s` |
| 泳道查询 `n=200,c=100` | `0` 失败，P95 `657ms` | 达到 `>=100` 并发目标 |

补充说明：

- 本轮压测中 Redis 压测后保留 `2` 个业务缓存键，说明查询缓存已生效
- 矩阵与泳道的 50/100 并发测试均未出现 5xx 或失败请求
- 指标满足 PRD 目标，但结果仅代表本地单机基线，不等同于正式生产容量评估

## 5. 风险记录

- `backend/tests/` 目录当前没有实际测试用例，自动化覆盖不足
- 飞书链路强依赖环境变量与飞书开放平台配置，配置错误会直接表现为登录失败
- `/api/export/jobs` 相关接口为占位实现，不可按已完成功能验收
- 当前性能结果基于本地单机环境，不代表公网网络、共享资源与生产数据波动下的极限表现

## 6. 建议结论

- 后端已达到“可联调、可继续验收”的状态
- 正式验收前应补充至少一轮接口级冒烟测试
- 如进入持续交付阶段，建议优先补充 `pytest + httpx.AsyncClient` 接口测试
- 建议在预发/生产环境按相同脚本再执行一轮压测，作为正式验收附件
