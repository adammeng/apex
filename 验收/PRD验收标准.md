# Apex V1 PRD 验收标准

## 1. 验收依据

- `需求/Apex早期靶点情报分析智能体PRD.docx`
- `需求/Apex早期靶点情报分析智能体PRD.pdf`

PRD 明确的本期范围如下：

- 飞书 SSO 登录集成
- 树状筛选侧边栏
- 靶点组合竞争矩阵视图
- 靶点研发进展泳道图视图
- 结构化数据导出至 Excel

以下内容不纳入本期验收：

- 数据录入/编辑
- AI 自动撰写分析报告
- 靶点视图 V2.0
- 管线历史事件视图 V2.0

## 2. 验收对象

- 前端：飞书入口、登录守卫、矩阵页、泳道页、导出交互
- 后端：鉴权接口、元数据接口、矩阵接口、泳道接口、导出接口、系统接口
- 部署：HTTPS 访问、飞书客户端访问、Docker 启动能力

## 3. 功能验收标准

### 3.1 飞书 SSO 登录

验收标准：

- 站点需支持飞书客户端内访问
- 首次访问时可通过飞书 SDK 获取授权码
- 后端可完成 `code -> JWT` 交换
- 成功登录后可通过 `/api/auth/me` 获取用户信息
- 非飞书浏览器访问时，应有明确引导策略

通过判定：

- 飞书客户端内可进入业务页
- 非飞书浏览器不会误进入未授权业务页

### 3.2 树状筛选侧边栏

验收标准：

- 支持治疗领域与疾病两级结构
- 支持多选、父子联动、半选状态
- 支持搜索与长文本展示
- 重置后恢复默认状态

### 3.3 竞争矩阵视图

验收标准：

- 基于最新记录规则输出矩阵
- 目标轴按分值与字典序排序
- 展示单靶点最高阶段与双靶点组合最高阶段
- 支持 tooltip 查看药品详情
- 支持“隐藏无组合靶点”
- 支持按治疗领域、疾病、阶段筛选

### 3.4 泳道图视图

验收标准：

- 单疾病必选
- 靶点支持多选与搜索
- 泳道展示完整阶段列
- 单元格直接展示药品信息
- 无数据时有明确空态

### 3.5 Excel 导出

验收标准：

- 矩阵与泳道图均可导出
- 导出文件字段与 PRD 定义一致
- 导出文件能被正常打开

### 3.6 系统与部署

验收标准：

- 服务可通过 HTTPS 访问
- 健康检查接口返回正常
- Docker 可完成核心服务启动
- 生产环境前端静态资源与后端 API 可同时访问

## 4. 非功能验收标准

### 4.1 可维护性

- 关键模块有 README 或架构文档支撑
- 配置项清晰，可区分开发与生产环境

### 4.2 可调试性

- 飞书登录失败时可定位到前端 SDK 或后端飞书接口错误
- 前端可输出运行时快照与授权调试信息
- 后端可返回细化后的飞书错误信息

### 4.3 质量基线

- 前端生产构建通过
- 后端核心模块语法检查通过
- PRD In Scope 范围内功能有对应实现与验证记录

### 4.4 性能验收基线

依据 PRD 7.1，V1 验收需关注以下量化目标：

- 矩阵渲染：`< 500ms`（50×50 规模）
- 数据查询：`< 5s`（复杂多条件筛选）
- 视图切换：`< 1s`（矩阵 ↔ 泳道图）
- 并发能力：`>= 100` 并发用户

其中后端验收重点为：

- `POST /api/matrix/query`、`POST /api/pipeline/query` 满足查询耗时目标
- Redis 缓存启用后，相同条件重复查询性能有明显收益
- 在压测条件下，核心接口不出现明显崩溃或大面积 5xx
